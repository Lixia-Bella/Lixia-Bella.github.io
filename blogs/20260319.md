# 阳光小站 CSS 动画与响应式优化实战总结

> 作者：Lixia ｜ 日期：2026-03-19

本文记录了阳光小站在 CSS 动画、响应式设计和暗色模式方面的一系列优化实践，涵盖导航栏交互、页面入场动画、断点适配以及主题切换四个核心模块。

## 一、导航栏链接高级 Hover 动画

为导航栏链接添加了基于 `::after` 伪元素的下划线滑入动画。核心思路是将伪元素初始宽度设为 0、定位在链接中央，hover 时通过 `width` 和 `left` 的过渡实现从中间向两侧展开的效果。使用 `cubic-bezier(0.25, 0.8, 0.25, 1)` 缓动函数让动画更加自然流畅。

当前激活页面的导航链接则始终显示下划线，采用奶油黄色配合 `box-shadow` 发光效果，与普通 hover 的白色下划线形成视觉区分。

移动端汉堡菜单展开时，通过 `@keyframes navItemSlideIn` 动画配合 `animation-delay` 递增（0.05s 步长），让每个菜单项从左侧依次错落滑入，营造出层次感。

## 二、页面卡片入场动画

定义了 `@keyframes slideUpFadeIn` 关键帧动画，元素从下方 40px 处淡入到原始位置。配合 Intersection Observer API，实现了滚动触发动画——只有当元素进入视口时才播放，且只播放一次。

通过 CSS 变量 `--delay` 控制每个元素的延迟时间，同类多卡片（如爱好卡片、关卡卡片）通过 `step` 参数实现递增延迟，产生错落入场的视觉效果。同时添加了 `prefers-reduced-motion` 媒体查询，尊重用户的动画偏好设置。

## 三、响应式断点优化

### clamp() 平滑字体缩放

将所有关键标题的固定 `font-size` 改为 `clamp()` 函数，例如页面主标题使用 `clamp(1.2rem, 4vw + 0.5rem, 1.9rem)`，字体在 320px 到 1024px 之间平滑过渡，无需在每个断点手动覆盖。

### 爱好卡片列数自适应

明确控制爱好卡片网格在不同屏幕下的列数：桌面端 3 列、平板（768px）2 列、手机（480px）1 列，替代了原来 `auto-fill` 的不精确行为。

### 导航栏平滑高度过渡

将移动端菜单的展开方式从 `transform: translateY` 改为基于 `max-height` + `padding` 的过渡动画，菜单展开时有自然的高度增长效果。

### 320px 小屏幕适配

针对极小屏幕新增了大量适配：隐藏品牌文字、缩小头像、减小内边距、积分栏纵向排列、规则网格单列等。

## 四、暗色模式切换

### 实现方案

采用 `data-theme="dark"` 属性 + CSS 变量覆盖的方案。在 `:root` 定义亮色变量，在 `[data-theme="dark"]` 中覆盖为暗色值。主色调降低饱和度，背景改为深紫灰色系，文字颜色反转为浅色。

### 防闪烁处理

在 `<script>` 标签执行前通过 IIFE 立即读取 `localStorage` 中的主题偏好，在 DOM 解析阶段就设置好 `data-theme` 属性，避免页面加载时的白屏闪烁。

### 切换动画

为所有主要元素添加了 `transition: background-color 0.4s, color 0.4s, border-color 0.4s, box-shadow 0.4s`，切换主题时颜色平滑过渡。导航栏中的切换按钮内含太阳/月亮双图标，通过 `opacity` 和 `transform: rotate()` 实现旋转切换动画。

### 系统偏好兼容

当用户未手动选择主题时，自动检测 `prefers-color-scheme: dark` 媒体查询，跟随系统暗色模式设置。
