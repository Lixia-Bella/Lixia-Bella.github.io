### Safe Web Fonts（安全网页字体）

#### 什么是 Safe Web Fonts？

**Safe Web Fonts**（安全网页字体）是指在绝大多数操作系统和浏览器中都预装的字体。使用这些字体可以确保网页在不同设备上都能正常显示，不会因为用户没有安装特定字体而出现显示问题。

#### 为什么需要安全网页字体？

当你在 CSS 中指定字体时，如果用户的设备没有安装该字体，浏览器会使用默认字体替代，这可能导致：

- 页面显示效果与设计不符
- 布局错乱
- 用户体验下降

#### 常见的安全网页字体

##### 衬线字体（Serif）


| 字体名称            | 适用系统                | CSS 使用示例                                 |
| --------------- | ------------------- | ---------------------------------------- |
| Georgia         | Windows、macOS、Linux | `font-family: Georgia, serif;`           |
| Times New Roman | Windows、macOS       | `font-family: "Times New Roman", serif;` |


##### 无衬线字体（Sans-serif）


| 字体名称         | 适用系统                | CSS 使用示例                                     |
| ------------ | ------------------- | -------------------------------------------- |
| Arial        | Windows、macOS、Linux | `font-family: Arial, sans-serif;`            |
| Helvetica    | macOS、iOS           | `font-family: Helvetica, Arial, sans-serif;` |
| Verdana      | Windows、macOS、Linux | `font-family: Verdana, sans-serif;`          |
| Tahoma       | Windows、macOS       | `font-family: Tahoma, sans-serif;`           |
| Trebuchet MS | Windows、macOS       | `font-family: "Trebuchet MS", sans-serif;`   |


##### 等宽字体（Monospace）


| 字体名称        | 适用系统                | CSS 使用示例                                 |
| ----------- | ------------------- | ---------------------------------------- |
| Courier New | Windows、macOS、Linux | `font-family: "Courier New", monospace;` |
| Consolas    | Windows             | `font-family: Consolas, monospace;`      |
| Monaco      | macOS               | `font-family: Monaco, monospace;`        |


#### 最佳实践

##### 1. 使用字体栈（Font Stack）

始终提供备选字体，以 `serif`、`sans-serif`、`monospace` 结尾：

```css
/* 无衬线字体栈 */
body {
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
}

/* 衬线字体栈 */
h1 {
  font-family: Georgia, "Times New Roman", Times, serif;
}

/* 等宽字体栈 */
code {
  font-family: Consolas, Monaco, "Courier New", monospace;
}
```

##### 2. 系统字体栈

使用系统原生字体可以获得最佳性能和原生体验：

```css
/* macOS / iOS 系统字体 */
font-family: -apple-system, BlinkMacSystemFont, "SF Pro Display";

/* Windows 系统字体 */
font-family: "Segoe UI", Tahoma, Geneva, Verdana, sans-serif;

/* 跨平台系统字体栈 */
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, "Noto Sans", sans-serif;
```

##### 3. Web Fonts 替代方案

如果需要使用非安全字体，可以使用 Web Fonts 服务：

- **Google Fonts**：[https://fonts.google.com/](https://fonts.google.com/)
- **Adobe Fonts**：[https://fonts.adobe.com/](https://fonts.adobe.com/)

```html
<!-- 引入 Google Fonts -->
<link href="https://fonts.googleapis.com/css2?family=Roboto&display=swap" rel="stylesheet">
```

```css
body {
  font-family: "Roboto", Arial, sans-serif;
}
```

### CSS 选择器

CSS 选择器用于选中 HTML 元素并应用样式。以下是 CSS 选择器的完整分类。

#### 基本选择器

##### 通配符选择器

选中所有元素。

```css
* {
	margin: 0;
	padding: 0;
}
```

##### 元素选择器（标签选择器）

选中所有指定标签的元素。

```css
p {
	color: #333;
}

h1 {
	font-size: 24px;
}
```

##### 类选择器

选中所有指定类名的元素，使用 `.` 开头。

```css
.container {
	width: 100%;
	max-width: 1200px;
}

.text-center {
	text-align: center;
}
```

```html
<div class="container">
	<p class="text-center">居中的文本</p>
</div>
```

##### ID 选择器

选中指定 ID 的元素，使用 `#` 开头。ID 在页面中应唯一。

```css
#header {
	background-color: #fff;
	position: fixed;
}

#main-content {
	padding: 20px;
}
```

```html
<header id="header">页面头部</header>
<main id="main-content">主要内容</main>
```

##### 基本选择器对比

| 选择器 | 语法     | 特点             | 使用场景     |
| ------ | -------- | ---------------- | ------------ |
| 通配符 | `*`      | 选中所有元素     | 重置样式     |
| 元素   | `tag`    | 选中所有同名标签 | 基础样式     |
| 类     | `.class` | 可复用，可多个   | 最常用       |
| ID     | `#id`    | 唯一性，权重高   | 页面唯一元素 |

#### 关系选择器

##### 后代选择器（空格）

选中某元素内部的所有指定后代元素（包括子元素、孙元素等）。

```css
/* 选中 article 内部的所有 p 元素 */
article p {
	line-height: 1.6;
}

/* 选中 nav 内部的所有 a 元素 */
nav a {
	text-decoration: none;
}
```

##### 子代选择器（`>`）

只选中某元素的直接子元素。

```css
/* 只选中 ul 的直接子元素 li，不包括嵌套的 li */
ul > li {
	list-style: none;
}

/* 只选中 nav 的直接子元素 a */
nav > a {
	padding: 10px;
}
```

##### 相邻兄弟选择器（`+`）

选中紧接在某元素之后的第一个兄弟元素。

```css
/* 选中紧跟在 h1 后面的第一个 p 元素 */
h1 + p {
	font-size: 18px;
	color: #666;
}
```

##### 通用兄弟选择器（`~`）

选中某元素之后的所有指定兄弟元素。

```css
/* 选中 h2 后面的所有 p 兄弟元素 */
h2 ~ p {
	margin-top: 10px;
}
```

##### 关系选择器对比

| 选择器   | 语法    | 选中范围                 |
| -------- | ------- | ------------------------ |
| 后代     | `A B`   | A 内部所有 B（包括嵌套） |
| 子代     | `A > B` | A 的直接子元素 B         |
| 相邻兄弟 | `A + B` | A 之后紧邻的第一个 B     |
| 通用兄弟 | `A ~ B` | A 之后的所有兄弟 B       |

#### 属性选择器

根据元素的属性及属性值来选中元素。

```css
/* 选中所有有 title 属性的元素 */
[title] {
	cursor: help;
}

/* 选中 type="text" 的 input */
input[type="text"] {
	border: 1px solid #ccc;
}

/* 选中 href 以 "https" 开头的 a 元素 */
a[href^="https"] {
	color: green;
}

/* 选中 href 以 ".pdf" 结尾的 a 元素 */
a[href$=".pdf"] {
	background: url("pdf-icon.png") no-repeat;
}

/* 选中 class 包含 "btn" 的元素 */
[class*="btn"] {
	padding: 10px 20px;
}

/* 选中 lang 属性以 "zh" 开头的元素（完整单词或连字符） */
[lang|="zh"] {
	quotes: "「" "」";
}
```

##### 属性选择器语法汇总

| 语法              | 说明                              |
| ----------------- | --------------------------------- |
| `[attr]`          | 有 attr 属性                      |
| `[attr="value"]`  | attr 属性值等于 value             |
| `[attr^="value"]` | attr 属性值以 value 开头          |
| `[attr$="value"]` | attr 属性值以 value 结尾          |
| `[attr*="value"]` | attr 属性值包含 value             |
| `[attr~="value"]` | attr 属性值包含 value（完整单词） |
| `[attr            | ="value"]`                        |

#### 伪类选择器

伪类用于选中元素的特定状态。

##### 状态伪类

/* 未访问的链接 */

#### 选择器权重（优先级）

当多个选择器选中同一元素时，权重高的生效。

| 选择器               | 权重               | 示例                                 |
| -------------------- | ------------------ | ------------------------------------ |
| `!important`         | 最高（不建议使用） | `color: red !important;`             |
| 行内样式             | 1000               | `style="color: red;"`                |
| ID 选择器            | 100                | `#header`                            |
| 类、伪类、属性选择器 | 10                 | `.active`、`:hover`、`[type="text"]` |
| 元素、伪元素选择器   | 1                  | `div`、`::before`                    |
| 通配符               | 0                  | `*`                                  |

**权重计算示例：**

```css
/* 权重: 1 */
p {
	color: black;
}

/* 权重: 10 */
.text {
	color: blue;
}

/* 权重: 100 */
#intro {
	color: green;
}

/* 权重: 1 + 10 = 11 */
p.text {
	color: red;
}

/* 权重: 100 + 10 = 110 */
#intro.text {
	color: orange;
}
```

> **提示**：尽量使用低权重选择器，避免使用 `!important`，保持样式可维护性。

### HTML 引用方式

所有 HTML 文件使用 `<script type="module">` 引入对应入口模块，同时保留 `<script nomodule>` 降级兼容旧浏览器：

```html
<!-- 现代浏览器使用 ES6 模块 -->
<script type="module" src="js/index.js"></script>
<!-- 旧浏览器降级到原有脚本 -->
<script nomodule src="scripts/main.js"></script>
<script nomodule src="scripts/blog.js"></script>
```

### 关键设计决策

1. **`scripts/` 旧文件保留不删除** — 通过 `nomodule` 属性实现向后兼容，现代浏览器会忽略 `nomodule` 脚本
2. **`scrollToTop` 挂载到 `window`** — 因为 HTML 中可能通过 `onclick="scrollToTop()"` 调用，module 作用域内的函数默认不在全局
3. **`blog-page.js` 使用 `waitForLibs()` 轮询** — 因为 `marked.js` 和 `highlight.js` 是普通 `<script>` 加载的 CDN 库，module 脚本执行时它们可能尚未就绪
4. **`quiz.js` 导入 `showToast`** — 体现模块间依赖的正确使用，避免重复定义

## Promise 并发的多种方法

### 1) `Promise.all()`

**特点**：全部成功才成功；任意一个失败会立即进入 `catch`。  
**适用场景**：多个请求结果都必需。

```js
const [user, posts] = await Promise.all([
    fetch("/api/user").then((res) => res.json()),
    fetch("/api/posts").then((res) => res.json())
]);
```

---

### 2) `Promise.allSettled()`

**特点**：等待全部任务结束（无论成功/失败），返回每个任务状态。  
**适用场景**：批量任务，允许部分失败。

```js
const results = await Promise.allSettled([
    fetch("/a"),
    fetch("/b"),
    fetch("/c")
]);

results.forEach((item) => {
    if (item.status === "fulfilled") {
        console.log("成功：", item.value);
    } else {
        console.log("失败：", item.reason);
    }
});
```

---

### 3) `Promise.race()`

**特点**：谁先完成（成功或失败）就返回谁。  
**适用场景**：超时控制、抢最快响应。

```js
const timeoutPromise = new Promise((_, reject) => {
    setTimeout(() => reject(new Error("请求超时")), 3000);
});

const data = await Promise.race([
    fetch("/api/data").then((res) => res.json()),
    timeoutPromise
]);
```

---

### 4) `Promise.any()`

**特点**：谁先成功就返回谁；全部失败才抛出 `AggregateError`。  
**适用场景**：多镜像地址容灾，只要一个成功即可。

```js
const fastestAvailable = await Promise.any([
    fetch("https://cdn1.example.com/data.json"),
    fetch("https://cdn2.example.com/data.json"),
    fetch("https://cdn3.example.com/data.json")
]);
```

---

### 5) 并发数限制（手动并发池）

**特点**：控制同一时间并发数量，防止请求过多压垮服务。  
**适用场景**：批量上传、批量下载、批量调用接口。

```js
const runWithLimit = async (tasks, limit = 3) => {
    const results = [];
    let index = 0;

    const workers = Array.from({ length: limit }).map(async () => {
        while (index < tasks.length) {
            const current = index++;
            try {
                results[current] = await tasks[current]();
            } catch (error) {
                results[current] = error;
            }
        }
    });

    await Promise.all(workers);
    return results;
};

// 使用示例：tasks 是函数数组，每个函数返回 Promise
// const resultList = await runWithLimit(tasks, 5);
```

---

### 6) 选型速记

- **全部都要成功**：`Promise.all`
- **要看每个任务结果**：`Promise.allSettled`
- **只关心最快结果（含失败）**：`Promise.race`
- **只要一个成功即可**：`Promise.any`
- **任务很多要控流**：并发池（限制并发数）

## React 组件化与 CSS 类名组合机制

在封装 React 组件时，经常会遇到动态拼接 CSS 类名的情况，例如：

```jsx
// Button.jsx
function Button({ text, type = 'default' }) {
  // 当 type 为 'default' 时，className 的值为 'btn btn-default'
  return <button className={`btn btn-${type}`}>{text}</button>;
}
```

而在对应的 CSS 文件中，我们通常是分开定义样式的：

```css
/* Button.css */
/* 基础按钮样式：负责排版、间距、动画等通用属性 */
.btn {
  padding: 8px 16px;
  border-radius: 4px;
}

/* 类型样式：只负责差异化的属性，如颜色、边框 */
.btn-default {
  background-color: #f0f0f0;
  color: #333;
}
```

### 为什么 CSS 中不需要写成 `.btn.btn-default`？

1. **HTML 类名的多重性**：
   在 HTML 中，`class="btn btn-default"` 表示该元素**同时拥有两个独立的类名**（用空格隔开）。
2. **CSS 样式的合并**：
   浏览器解析时，会把 `.btn` 的样式和 `.btn-default` 的样式**合并**应用到这个按钮上。
3. **分离关注点（OOCSS 思想）**：
   - `.btn` 提取了所有按钮的**共性**（高内聚）。
   - `.btn-default`、`.btn-primary` 等提取了按钮的**特性**（低耦合）。
   - 这种设计模式使得 CSS 代码复用率极高，修改基础样式时只需改 `.btn` 一处即可，不需要在每个类型里重复写 `padding`、`border-radius` 等属性。
4. **选择器的独立性**：
   在 CSS 中，`.btn-default` 是一个独立的选择器，只要 HTML 元素身上带有这个类名，它的样式就会生效，无需强制它必须和 `.btn` 绑定在一起写。

## React 核心概念：Props 与 State 的区别

在 React 中，`Props` 和 `State` 都是用于控制组件数据和渲染的核心概念，但它们有着本质的区别。

### 1. Props (属性)
- **定义**：组件的只读属性，类似于函数的参数。
- **来源**：由父组件传递给子组件。
- **可变性**：**只读（Read-only）**。子组件绝对不能直接修改接收到的 Props。
- **用途**：用于组件间的数据传递和配置（例如：配置 Button 的颜色、传递 Card 的标题）。

### 2. State (状态)
- **定义**：组件内部维护的私有数据，可以理解为组件的“记忆”。
- **来源**：在组件内部声明和初始化（在函数组件中使用 `useState` Hook）。
- **可变性**：**可变（Mutable）**。组件可以通过特定的更新函数（如 `setExpanded`）来修改自己的状态。
- **用途**：用于管理组件内部的交互和动态数据（例如：控制卡片的展开/收起、输入框的值、点赞数）。**状态的改变会自动触发组件的重新渲染**。

### 3. 核心对比总结

| 特性 | Props (属性) | State (状态) |
| :--- | :--- | :--- |
| **数据流向** | 从父组件流向子组件 (自上而下) | 组件内部自己管理 |
| **修改权限** | **不可修改** (只读) | **可以修改** (通过 setState) |
| **主要作用** | 组件间通信、复用配置 | 响应用户交互、控制内部 UI 变化 |
| **类比** | 别人给你的指令/配置 | 你自己的记忆/心情 |

**💡 一句话总结**：
Props 是**外部传入**的，组件无法改变它；State 是**内部私有**的，组件可以自己改变它。如果一个数据需要随时间或用户交互而改变，它应该作为 State；如果一个数据只是用来展示或作为初始配置，它应该作为 Props。

## 第一周：前端核心与 React 基础

### 第五天：React Hooks 进阶与数据持久化

#### 1. useState 进阶
- `useState` 不仅可以存储基本数据类型（如数字、字符串、布尔值），还可以存储对象和数组。
- **更新对象/数组状态时的注意事项**：React 中的状态被认为是不可变的（immutable）。在更新对象或数组时，不能直接修改原对象，而是应该创建一个新的对象/数组。通常使用 ES6 的展开运算符（`...`）来复制原有属性，并覆盖需要修改的部分。
  ```jsx
  const [user, setUser] = useState({ name: '张三', age: 25 });
  // 正确的更新方式：
  setUser({ ...user, age: 26 });
  ```
- **惰性初始化**：如果初始状态需要通过复杂计算或读取 `localStorage` 获得，可以给 `useState` 传入一个函数。这个函数只会在组件初次渲染时执行一次，从而提升性能。
  ```jsx
  const [count, setCount] = useState(() => {
    const saved = localStorage.getItem('myCount');
    return saved ? parseInt(saved, 10) : 0;
  });
  ```

#### 2. useEffect 依赖项数组的不同用法与注意事项
`useEffect` 用于在函数组件中执行副作用操作（如数据获取、DOM 操作、订阅等）。它的第二个参数是**依赖项数组**，决定了副作用函数何时重新执行。

**不同用法的总结：**

| 依赖项数组写法 | 触发时机 | 适用场景 | 类似生命周期 |
| :--- | :--- | :--- | :--- |
| **不传第二个参数**<br/>`useEffect(() => {...})` | 组件**每次渲染后**都会执行。 | 极少使用。如果内部修改了状态，极易导致无限循环渲染。 | `componentDidMount` + `componentDidUpdate` |
| **空数组 `[]`**<br/>`useEffect(() => {...}, [])` | 只在组件**初次挂载（挂载后）**执行一次。 | 初始化操作：如发送网络请求获取初始数据、设置全局事件监听器、开启定时器。 | `componentDidMount` |
| **包含特定变量 `[A, B]`**<br/>`useEffect(() => {...}, [A, B])` | 初次挂载时执行一次，之后**当 A 或 B 的值发生变化时**重新执行。 | 响应状态变化：如根据搜索词重新请求数据、将更新后的状态同步到 `localStorage`。 | `componentDidUpdate` (带条件) |

**注意事项（避坑指南）：**
1. **清理副作用（Cleanup）**：
   如果副作用中包含了订阅、定时器或事件监听，必须在 `useEffect` 中返回一个清理函数，以防止内存泄漏。清理函数会在组件卸载前，以及下一次副作用执行前运行。
   ```jsx
   useEffect(() => {
     const timer = setInterval(() => console.log('tick'), 1000);
     // 返回清理函数
     return () => clearInterval(timer);
   }, []);
   ```
2. **依赖项必须诚实**：
   如果 `useEffect` 内部使用了组件作用域内的变量（如 state、props 或由它们衍生出的函数），这些变量**必须**包含在依赖项数组中。否则，副作用内部可能会拿到旧的（闭包陷阱中的）数据。
3. **避免无限循环**：
   如果在 `useEffect` 中更新了某个状态，而这个状态又在依赖项数组中，就会导致：状态更新 -> 触发渲染 -> 触发 Effect -> 再次更新状态... 的无限循环。解决办法通常是使用函数式更新（如 `setCount(c => c + 1)`）或仔细检查依赖逻辑。

#### 3. localStorage 数据持久化
- `localStorage` 是浏览器提供的 API，数据保存在本地，刷新或关闭页面不会丢失。
- **存储数据**：`localStorage.setItem('key', 'value')`。注意，value 必须是字符串。如果是对象或数组，需要用 `JSON.stringify()` 转换。
- **读取数据**：`localStorage.getItem('key')`。如果是 JSON 字符串，需要用 `JSON.parse()` 转换回对象。
- **结合 React**：通常在组件初始化时（通过 `useState` 的惰性初始化函数）读取 `localStorage`，并使用 `useEffect` 监听状态变化，实时将其写入 `localStorage`。
